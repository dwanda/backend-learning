🚀 手把手教你用 Spring Boot 开发一个 CRUD 应用

本教程专为前端开发者设计——你已经有编程基础，只需要补齐后端这一块。 我们将从零开始，一步步搭建一个完整的 用户管理 CRUD 项目。

学完之后你能做什么：独立搭建 Spring Boot 项目，连接 MySQL 数据库，写出完整的 RESTful CRUD API。 这就像前端学会了"搭项目 → 写组件 → 调接口"一样，是后端开发的驾照。

4.2 写入 MySQL 连接配置

src/main/resources/application.yml

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/learning?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
    username: root
    password: root123
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: update       # 自动根据 Entity 创建/更新表结构（开发用，生产不要用）
    show-sql: true           # 控制台打印 SQL，方便调试
    properties:

ddl-auto: update 会根据你的 Entity 类自动建表/加字段，省去了手写 DDL 的麻烦。

5创建数据实体 Entity

Entity = 数据库表对应的 Java 类。 一个 Entity 实例 = 表中的一行数据。 通过 JPA 注解，你可以告诉 Spring Boot 这个类对应哪张表、字段对应哪些列。

在 IDEA 中创建新包 com.example.demo.entity，然后创建 User.java：

src/main/java/com/example/demo/entity/User.java

package com.example.demo.entity;

import jakarta.persistence.*;
import lombok.*;
import java.time.LocalDateTime;

@Entity                    // 标记这是一个 JPA 实体
@Table(name = "users")     // 对应数据库中的 users 表
@Data                      // Lombok：自动生成 getter/setter/toString/hashCode/equals
@NoArgsConstructor         // 无参构造器（JPA 需要）
@AllArgsConstructor        // 全参构造器（方便创建对象）
@Builder                   // Builder 模式
public class User {

    @Id                      // 主键
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // 自增
    private Long id;

    @Column(nullable = false, length = 50)
    private String username;

    @Column(nullable = false, unique = true, length = 100)
    private String email;

    @Column(name = "phone_number", length = 20)  // 指定列名（Java 驼峰 → DB 下划线）
    private String phoneNumber;

    @Column(nullable = false)
    private Integer age;

    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @Column(name = "updated_at")
    private LocalDateTime updatedAt;

    // 新增前自动设置创建时间
    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    // 更新前自动刷新更新时间
    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }
}

注解速查表

JPA 注解作用
@Entity告诉 JPA：这是个实体类，要映射到数据库表
@Table(name="xxx")指定表名，不写默认用类名转下划线
@Id标记主键字段
@GeneratedValue主键生成策略，IDENTITY = 自增
@Column配置列属性（长度、非空、唯一等）
@PrePersist在 insert 之前自动执行的方法
@PreUpdate在 update 之前自动执行的方法
Lombok 注解自动生成什么
@Datagetter + setter + toString + equals + hashCode
@NoArgsConstructor无参构造器（JPA 的硬性要求）
@AllArgsConstructor包含所有字段的构造器
@BuilderBuilder 模式，方便构造对象
Lombok 让你不用手写 getter/setter/构造器。IDEA 需要装 Lombok 插件： Preferences → Plugins → 搜索 "Lombok" → Install。
另外还需要在 Settings → Build → Compiler → Annotation Processors 里勾选 Enable annotation processing。

6创建 Repository —— 零 SQL 操作数据库

Repository = 数据访问层。 Spring Data JPA 提供了强大到不可思议的 Repository 接口：你只需要定义一个接口并继承它，CRUD 方法就已经自动生成了。

创建包 com.example.demo.repository，然后创建 UserRepository.java：

src/main/java/com/example/demo/repository/UserRepository.java

package com.example.demo.repository;

import com.example.demo.entity.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

import java.util.Optional;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // JpaRepository 自带以下方法，你不用写一行代码：
    //   save(user)      → 新增/更新
    //   findById(id)    → 按主键查询
    //   findAll()       → 查全部
    //   deleteById(id)  → 按主键删除
    //   count()         → 总数
    //   existsById(id)  → 是否存在

    // 你只需要声明自定义的查询方法，JPA 会根据方法名自动生成 SQL！
    boolean existsByEmail(String email);

    boolean existsByUsername(String username);

    Optional<User> findByEmail(String email);
}

这是什么魔法？

JpaRepository<User, Long> 的两个泛型参数： User = 实体类，Long = 主键类型。

方法命名规则（Spring Data JPA 的核心魔法）： existsByEmail(String email) → JPA 自动生成 SQL：SELECT COUNT(*) FROM users WHERE email = ?

方法名前缀SQL 操作示例
findBySELECTfindByUsername
existsBySELECT COUNTexistsByEmail
countBySELECT COUNTcountByAgeGreaterThan
deleteByDELETEdeleteByUsername
这就是 Spring Data JPA 最强大的特性：约定优于配置。方法名写对了，SQL 就自动生成。只有复杂查询才需要手写 @Query。

7创建 Service —— 业务逻辑层

Service = 业务逻辑层。 它是 Controller 和 Repository 之间的桥梁： Controller 负责接收 HTTP 请求，Service 负责处理业务逻辑，Repository 负责和数据库打交道。

为什么需要 Service？ 如果不分层，Controller 里既要处理 HTTP 又要写业务逻辑又要查数据库——代码会变成一坨。分层让每一层职责清晰、可测试、可替换。

创建包 com.example.demo.dto，先定义请求和响应的 DTO：

src/main/java/com/example/demo/dto/CreateUserRequest.java

package com.example.demo.dto;

import lombok.Data;

@Data
public class CreateUserRequest {
    private String username;
    private String email;
    private String phoneNumber;
    private Integer age;
}

src/main/java/com/example/demo/dto/UpdateUserRequest.java

package com.example.demo.dto;

import lombok.Data;

@Data
public class UpdateUserRequest {
    private String username;
    private String email;
    private String phoneNumber;
    private Integer age;
}

src/main/java/com/example/demo/dto/UserResponse.java

package com.example.demo.dto;

import com.example.demo.entity.User;
import lombok.Builder;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@Builder
public class UserResponse {
    private Long id;
    private String username;
    private String email;
    private String phoneNumber;
    private Integer age;
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    // Entity → Response DTO 的转换方法
    public static UserResponse fromEntity(User user) {
        return UserResponse.builder()
            .id(user.getId())
            .username(user.getUsername())
            .email(user.getEmail())
            .phoneNumber(user.getPhoneNumber())
            .age(user.getAge())
            .createdAt(user.getCreatedAt())
            .updatedAt(user.getUpdatedAt())
            .build();
    }
}
为什么要分开 Request 和 Response DTO？ Entity 是和数据库对应的，不应该直接暴露给前端。 Request DTO 只包含前端要提交的字段；Response DTO 只包含前端需要看的字段。 这叫做"输入输出分离"，是后端开发的最佳实践。

现在创建包 com.example.demo.service 和 UserService.java：

src/main/java/com/example/demo/service/UserService.java

package com.example.demo.service;

import com.example.demo.dto.*;
import com.example.demo.entity.User;
import com.example.demo.repository.UserRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.data.domain.*;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor  // Lombok：为 final 字段自动生成构造器 → Spring 自动注入
public class UserService {

    private final UserRepository userRepository;

    // ========== 创建用户 ==========
    @Transactional
    public UserResponse createUser(CreateUserRequest request) {
        // 检查用户名和邮箱唯一性
        if (userRepository.existsByUsername(request.getUsername())) {
            throw new RuntimeException("用户名已存在：" + request.getUsername());
        }
        if (userRepository.existsByEmail(request.getEmail())) {
            throw new RuntimeException("邮箱已存在：" + request.getEmail());
        }

        // DTO → Entity
        User user = new User();
        user.setUsername(request.getUsername());
        user.setEmail(request.getEmail());
        user.setPhoneNumber(request.getPhoneNumber());
        user.setAge(request.getAge());

        // 保存到数据库（@PrePersist 会自动设创建时间）
        User savedUser = userRepository.save(user);

        // Entity → Response DTO
        return UserResponse.fromEntity(savedUser);
    }

    // ========== 查询单个用户 ==========
    public UserResponse getUserById(Long id) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("用户不存在，id：" + id));
        return UserResponse.fromEntity(user);
    }

    // ========== 分页查询 ==========
    public Page<UserResponse> getAllUsers(int page, int size) {
        Pageable pageable = PageRequest.of(page, size, Sort.by("createdAt").descending());
        return userRepository.findAll(pageable)
            .map(UserResponse::fromEntity);
    }

    // ========== 更新用户 ==========
    @Transactional
    public UserResponse updateUser(Long id, UpdateUserRequest request) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("用户不存在，id：" + id));

        // 只更新非空字段
        if (request.getUsername() != null) user.setUsername(request.getUsername());
        if (request.getEmail() != null) user.setEmail(request.getEmail());
        if (request.getPhoneNumber() != null) user.setPhoneNumber(request.getPhoneNumber());
        if (request.getAge() != null) user.setAge(request.getAge());

        // save() 方法：如果对象有 id 且数据库中已存在，就是 UPDATE；否则是 INSERT
        // @PreUpdate 会自动刷新 updatedAt
        User savedUser = userRepository.save(user);
        return UserResponse.fromEntity(savedUser);
    }

    // ========== 删除用户 ==========
    @Transactional
    public void deleteUser(Long id) {
        if (!userRepository.existsById(id)) {
            throw new RuntimeException("用户不存在，id：" + id);
        }
        userRepository.deleteById(id);
    }
}

关键注解说明

注解作用
@Service标记为 Service Bean，Spring 会自动扫描并管理它
@RequiredArgsConstructorLombok：为 final 字段生成构造器。Spring 会自动注入构造器参数——这叫构造器注入，是目前推荐的做法
@Transactional方法内的数据库操作在同一个事务中执行。要么全部成功，要么全部回滚
关于依赖注入（DI）：注意 UserRepository 是一个 interface，我们并没有写它的实现类。 Spring Data JPA 在运行时自动生成了代理实现类并注入进来。这就是 Spring 的核心能力——控制反转（IoC）。
到这里，Service 层已经写好了。你能解释 Controller → Service → Repository 三层各自的职责吗？
（Controller：接收请求，返回响应；Service：业务逻辑；Repository：数据库操作）

8统一的返回格式

前端调用 API 时，期望得到的 JSON 格式应该是统一的。不能有的接口返回纯数据，有的返回带 success 字段的，有的返回字符串。 我们来定义一个统一的响应包装类 R<T>：

创建包 com.example.demo.common，然后创建 R.java：

src/main/java/com/example/demo/common/R.java

package com.example.demo.common;

import lombok.Data;

@Data
public class R<T> {
    private int code;      // 200 成功，4xx 客户端错误，5xx 服务端错误
    private String message;  // 提示信息
    private T data;         // 泛型数据（可以是 UserResponse、List、Page 等任意类型）

    private R(int code, String message, T data) {
        this.code = code;
        this.message = message;
        this.data = data;
    }

    // ========== 快捷静态工厂方法 ==========
    public static <T> R<T> ok(T data) {
        return new R<>(200, "success", data);
    }

    public static <T> R<T> ok() {
        return new R<>(200, "success", null);
    }

    public static <T> R<T> fail(int code, String message) {
        return new R<>(code, message, null);
    }
}

所有接口的返回格式现在统一为：

{
  "code": 200,
  "message": "success",
  "data": { ... }   // 实际数据
}
这个模式在后端工程里叫 统一响应体（Unified Response），前端可以统一判断 code === 200 来决定是否成功， 不用每个接口都写不同的判断逻辑。

9创建 Controller —— REST API 接口

Controller = 前端的"路由文件"。 它接收 HTTP 请求，调用 Service，把结果包装成统一格式返回。

创建包 com.example.demo.controller，然后创建 UserController.java：

src/main/java/com/example/demo/controller/UserController.java

package com.example.demo.controller;

import com.example.demo.common.R;
import com.example.demo.dto.*;
import com.example.demo.service.UserService;
import lombok.RequiredArgsConstructor;
import org.springframework.data.domain.Page;
import org.springframework.web.bind.annotation.*;

@RestController                          // = @Controller + @ResponseBody（返回值自动序列化为 JSON）
@RequestMapping("/users")             // 统一前缀：所有接口都在 /users 下
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    // ========== POST /users —— 创建用户 ==========
    @PostMapping
    public R<UserResponse> create(@RequestBody CreateUserRequest request) {
        UserResponse response = userService.createUser(request);
        return R.ok(response);
    }

    // ========== GET /users/{id} —— 查询用户详情 ==========
    @GetMapping("/{id}")
    public R<UserResponse> getById(@PathVariable Long id) {
        UserResponse response = userService.getUserById(id);
        return R.ok(response);
    }

    // ========== GET /users —— 分页列表 ==========
    @GetMapping
    public R<Page<UserResponse>> list(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size) {
        Page<UserResponse> responses = userService.getAllUsers(page, size);
        return R.ok(responses);
    }

    // ========== PUT /users/{id} —— 更新用户 ==========
    @PutMapping("/{id}")
    public R<UserResponse> update(
            @PathVariable Long id,
            @RequestBody UpdateUserRequest request) {
        UserResponse response = userService.updateUser(id, request);
        return R.ok(response);
    }

    // ========== DELETE /users/{id} —— 删除用户 ==========
    @DeleteMapping("/{id}")
    public R<Void> delete(@PathVariable Long id) {
        userService.deleteUser(id);
        return R.ok();
    }
}

Controller 注解速查

注解对应的 HTTP 方法类比前端
@GetMappingGET（查询）router.get('/users', ...)
@PostMappingPOST（新增）router.post('/users', ...)
@PutMappingPUT（全量更新）router.put('/users/:id', ...)
@DeleteMappingDELETE（删除）router.delete('/users/:id', ...)
参数注解来源示例
@PathVariableURL 路径参数/users/{id} → @PathVariable Long id

10统一异常处理

现在的代码里用 throw new RuntimeException("...") 来抛异常。 如果不做处理，前端会收到一个 500 错误和一段丑陋的堆栈信息。 我们用 @RestControllerAdvice 来统一拦截异常，返回友好的 JSON。

在 com.example.demo.common 包内创建异常类：

src/main/java/com/example/demo/common/BusinessException.java

package com.example.demo.common;

import lombok.Getter;

@Getter
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
}

在 com.example.demo.common 包内创建全局异常处理器：

src/main/java/com/example/demo/common/GlobalExceptionHandler.java

package com.example.demo.common;

import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.*;

@Slf4j                               // Lombok：自动生成 log 对象
@RestControllerAdvice               // 全局异常拦截器
public class GlobalExceptionHandler {

    // 拦截自定义 BusinessException
    @ExceptionHandler(BusinessException.class)
    public R<Void> handleBusinessException(BusinessException e) {
        return R.fail(e.getCode(), e.getMessage());
    }

    // 兜底：拦截所有未处理的异常
    @ExceptionHandler(Exception.class)
    public R<Void> handleException(Exception e) {
        log.error("未处理异常：", e);
        return R.fail(500, "服务器内部错误：" + e.getMessage());
    }
}

然后修改 UserService，把 RuntimeException 替换为 BusinessException：

// 之前：
throw new RuntimeException("用户名已存在：" + request.getUsername());

// 改成：
throw new BusinessException(400, "用户名已存在：" + request.getUsername());
@RestControllerAdvice 只拦截进入 Controller 之后抛出的异常。 如果在 Filter 或 Interceptor 里（Controller 之前）抛出异常，需要用其他方式处理。

11参数校验

目前 Controller 不会校验前端传来的参数是否合法——如果用户不填 username，Service 会报错， 但错误信息不够清晰。用 @Valid + Bean Validation 注解可以在请求进来时就校验。

11.1 给 DTO 加校验注解

src/main/java/com/example/demo/dto/CreateUserRequest.java（修改后）

package com.example.demo.dto;

import jakarta.validation.constraints.*;
import lombok.Data;

@Data
public class CreateUserRequest {

    @NotBlank(message = "用户名不能为空")
    @Size(min = 2, max = 50, message = "用户名长度需在 2-50 之间")
    private String username;

    @NotBlank(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;

    @Pattern(regexp = "^1[3-9]\\d{9}$", message = "手机号格式不正确")
    private String phoneNumber;

    @NotNull(message = "年龄不能为空")
    @Min(value = 0, message = "年龄不能为负数")
    @Max(value = 150, message = "年龄不能超过150")
    private Integer age;
}

11.3 全局异常处理器里捕获校验失败异常

在 GlobalExceptionHandler 里增加：

// 处理 @Valid 校验失败
@ExceptionHandler(MethodArgumentNotValidException.class)
public R<Void> handleValidation(MethodArgumentNotValidException e) {
    String message = e.getBindingResult().getFieldErrors()
        .stream()
        .map(fieldError -> fieldError.getField() + ": " + fieldError.getDefaultMessage())
        .collect(Collectors.joining(", "));
    return R.fail(400, message);
}

常用校验注解速查

注解说明
@NotNull不能是 null
@NotBlank不能是 null 且不能是空字符串（用于 String）
@NotEmpty不能是 null 且 size > 0（用于集合/String）
@Size(min, max)字符串长度或集合大小
@Min / @Max数值范围
@Email邮箱格式
@Pattern正则匹配

12分页查询

分页查询已经在 UserService.getAllUsers() 里实现了，这里回顾一下关键代码： 

public Page<UserResponse> getAllUsers(int page, int size) {
    // PageRequest.of(页码, 每页大小, 排序规则)
    Pageable pageable = PageRequest.of(page, size, Sort.by("createdAt").descending());
    return userRepository.findAll(pageable)
        .map(UserResponse::fromEntity);  // 把 Entity 的 Page 转成 Response 的 Page
}

返回的 JSON 结构类似：

{
  "code": 200,
  "message": "success",
  "data": {
    "content": [ ... ],       // 当前页数据
    "totalElements": 100,     // 总记录数
    "totalPages": 10,         // 总页数
    "number": 0,              // 当前页码（从 0 开始）
    "size": 10                // 每页大小
  }
}
.map(UserResponse::fromEntity) 用到了 Java 的方法引用语法，等价于 .map(entity -> UserResponse.fromEntity(entity))。 这是 Java 8+ 的函数式特性，和 JS 的 array.map(fn) 一模一样。

demo/
├── pom.xml
└── src/main/
    ├── java/com/example/demo/
    │   ├── DemoApplication.java              ← 启动入口
    │   ├── common/
    │   │   ├── R.java                         ← 统一响应包装
    │   │   ├── BusinessException.java         ← 业务异常
    │   │   └── GlobalExceptionHandler.java    ← 全局异常处理
    │   ├── controller/
    │   │   └── UserController.java            ← REST API 接口
    │   ├── dto/
    │   │   ├── CreateUserRequest.java         ← 新增请求
    │   │   ├── UpdateUserRequest.java         ← 更新请求
    │   │   └── UserResponse.java              ← 响应
    │   ├── entity/
    │   │   └── User.java                      ← 数据库实体
    │   ├── repository/
    │   │   └── UserRepository.java            ← 数据访问接口
    │   └── service/
    │       └── UserService.java               ← 业务逻辑
    └── resources/
        └── application.yml                    ← 数据库/服务器配置

这就是一个标准 Spring Boot 项目的三层分层结构：

  • controller/ — 接收请求、返回响应（不写逻辑）
  • service/ — 业务逻辑（不知道 HTTP 的存在）
  • repository/ — 数据库操作（一个接口 = SQL 语句）
  • entity/ — 数据库表对应的 Java 类
  • dto/ — 前后端传输的数据对象
  • common/ — 通用工具（异常、响应格式等）

15常见问题 FAQ

Q：启动报错 "Access denied for user 'root'@'localhost'"

A：MySQL 密码不对。检查 application.yml 里的 password 是否正确。

Q：启动报错 "Unknown database 'learning'"

A：忘了创建数据库。进 MySQL 执行 CREATE DATABASE learning DEFAULT CHARACTER SET utf8mb4;

Q：启动报错 "Table 'users' doesn't exist"

A：检查 ddl-auto 是否设为 update。启动后 JPA 会自动建表。

Q：Lombok 注解全部报红

A：IDEA 没装 Lombok 插件。Preferences → Plugins → 搜 "Lombok" → Install。并且 Settings → Build → Compiler → Annotation Processors → 勾选 "Enable annotation processing"。

Q：Entity 里加了字段但数据库表不更新

A：ddl-auto: update 只能加新字段，不会删字段、不会改字段类型。如果改动较大，把表删了让 JPA 重新建，或者以后用 Flyway 管理数据库变更。

Q：mvn spring-boot:run 太慢

A：第一次需要下载所有依赖（~200MB），之后就好了。在 IDEA 里点绿色 ▶ 启动更快（热加载）。
