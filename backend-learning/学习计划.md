# 后端学习计划 — 前端开发者视角

> 目标：以最快路径达到**能直接上手开发**的水平。
> 不用全部学，优先掌握开发中最常用的 20% 知识点就能覆盖 80% 的工作。

## 整体策略

这套技术栈东西很多。按**使用频率 + 开发刚需**排列，分阶段推进。**不必学完一阶段再进入下一阶段**——每个阶段的实践直接复用上一阶段的项目，迭代式叠加。

---

## 阶段一：环境搭建 + 第一条 API（预计 2-3 天）

### 必须掌握
| 知识点 | 要求 | 说明 |
|--------|------|------|
| JDK 21 安装 / IDEA 配置 | 能创建 Spring Boot 项目 | 用 [start.spring.io](https://start.spring.io) 生成项目 |
| Maven 基础 | 看懂 `pom.xml`，会 `mvn clean package` | 不深究生命周期，先会用 |
| Spring Boot 启动类 + `@SpringBootApplication` | 知道项目从哪启动 | |
| `@RestController` + `@GetMapping`/`@PostMapping` | 写一条 GET `/hello` 和 POST `/echo` | **第一个里程碑** |

### 直接练手项目
> 用一个项目贯穿所有阶段：**"用户管理"**

**阶段一成果**：一个 Spring Boot 项目，跑在 8080 端口，返回 JSON。

```java
@RestController
@RequestMapping("/users")
public class UserController {
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return new User(id, "测试用户");
    }
}
```

### 可跳过
- Maven 高级插件、多模块构建
- Spring Boot 自动配置原理
- 各种 Starter 的内部机制

## 阶段二：分层架构 + 数据库 CRUD（预计 3-5 天）

这是后端开发的**核心骨架**，这阶段完成后就可以参与大部分业务代码开发了。

### 必须掌握
| 知识点 | 要求 |
|--------|------|
| Controller → Service → Repository 三层分层 | 理解各层职责，这是 DDD 的简化版 |
| Spring Data JPA + `@Entity` | 写一个实体类映射到表 |
| JPA Repository 基本 CRUD（`save`, `findById`, `findAll`, `deleteById`） | 能完成增删改查 |
| `@Service` + `@Autowired` / 构造器注入 | 理解依赖注入 |
| `application.yml` 配置 MySQL 数据源 | 能连上 MySQL 8.0 |
| `@RequestBody` / `@RequestParam` / `@PathVariable` | 参数接收 |
| 统一返回结果包装 `R<T>` | 跟团队风格一致 |

### 直接练手项目（复用阶段一项目）
**用户管理 API**：实现 `UserController` + `UserService` + `UserRepository`

| 接口 | 说明 |
|------|------|
| `POST /users` | 创建用户 |
| `GET /users/{id}` | 查询用户详情 |
| `GET /users` | 分页列表 |
| `PUT /users/{id}` | 更新用户 |
| `DELETE /users/{id}` | 删除用户 |

### 可跳过 / 后学
- JPA 复杂关联映射（`@OneToMany`、`@ManyToMany`）—— 等遇到再查
- 事务传播行为——先只记 `@Transactional`
- QueryDSL / Specification —— 复杂度查询场景再用
- 连接池调优（HikariCP）

## 阶段三：DDD 入门 — 理解项目代码结构（预计 1-2 天）

> 团队用 DDD 思想，但你**不需要从头学 Evans 那本蓝皮书**。只需要理解项目中代码包怎么分层。

### 必须掌握的 DDD 概念
| 概念 | 对应代码位置 | 等价理解 |
|------|-------------|---------|
| Controller（接口层） | `controller/` | 接收请求、返回响应 |
| Application Service（应用层） | `application/` | 编排逻辑、事务脚本 |
| Domain（领域层） | `domain/` | 核心业务逻辑、实体、值对象 |
| Infrastructure（基础设施层） | `infrastructure/` | DB、MQ、外部服务 |
| Repository（仓储） | `domain/repository/` | 数据访问抽象（在 DDD 中不等于 DAO） |

**实操建议**：找一个现有的业务代码，按上面的目录结构找一遍对应关系。能说清每个类的职责即可，不需要写 DDD 代码。

### 可跳过
- 领域事件（Domain Event）—— 后面学
- 聚合（Aggregate）设计 —— 后面学
- CQRS、Event Sourcing

---

## 阶段四：Nacos + 配置管理（预计 1 天）

> Nacos 是这套技术栈的服务注册中心和配置中心。在本地开发中最常用的功能是**配置管理**。

### 必须掌握
| 知识点 | 要求 |
|--------|------|
| Nacos 本地启动（docker-compose） | 跑起来 |
| `bootstrap.yml` 配置 Nacos 地址 | 从 Nacos 拉配置 |
| Nacos 配置中心：新建 Data ID、发布配置 | 在界面上操作一次 |
| `@RefreshScope` + 动态刷新 | 在线改配置不重启 |
| Nacos 服务注册 + `@EnableDiscoveryClient` | 知道项目注册到 Nacos |

### 练手
- 在 Nacos 配置中心管理 `datasource.url` 等配置
- 把本地 `application.yml` 里的一些配置搬到 Nacos

### 可跳过
- Nacos 集群部署、AP/CP 模式切换
- Nacos 权限控制
- 服务发现的高级用法（负载均衡策略、权重）

## 阶段五：Redis 缓存（预计 1-2 天）

### 必须掌握
| 知识点 | 要求 |
|--------|------|
| Redis 基本数据类型（String、Hash） | 知道存什么样子 |
| `spring-boot-starter-data-redis` + `RedisTemplate` | 缓存查询结果 |
| `@Cacheable` / `@CacheEvict` 声明式缓存 | 最常用的方式 |
| 缓存常见问题：缓存穿透、雪崩 | 知道概念和简单处理 |

### 练手
在用户管理项目中，给 `GET /users/{id}` 加缓存：
```java
@Cacheable(value = "user", key = "#id")
public User getUser(Long id) {
    return userRepository.findById(id).orElseThrow(...);
}
```

### 可跳过
- Redis 持久化（RDB/AOF）、集群、哨兵
- Redis Lua 脚本
- 分布式锁 —— 后面用到再学

---

## 阶段六：RabbitMQ — 消息发送 & 消费（预计 1-2 天）

> 如果你之前用过前端的事件/消息机制（如 EventEmitter、PostMessage），RabbitMQ 本质是同样的**发布-订阅**模式。

### 必须掌握
| 知识点 | 要求 |
|--------|------|
| 核心概念：Producer / Consumer / Queue / Exchange / Routing Key | 能画出来这几个的关系 |
| 直连交换机（Direct Exchange） | 点对点消息 |
| 主题交换机（Topic Exchange） | 按规则路由消息 |
| `@RabbitListener` 消费消息 | 写一个消费者 |
| `RabbitTemplate.convertAndSend()` 发送消息 | 写一个生产者 |

### 练手
在用户管理项目中增加：用户注册后发送一条"新用户通知"消息，另一个服务收到后打印日志。

### 可跳过
- 死信队列（DLQ）、延迟队列 —— 需要时再查
- 消息确认机制（Confirm / Return）—— 先有概念
- RabbitMQ 集群、镜像队列、Shovel
- 消息幂等性 —— 等到有重复消费问题再解决

## 总结：优先级热力图

```
★★★★★ 立即学（阶段一二）   —— Spring Boot 分层 + DB CRUD
★★★★☆ 尽快学（阶段三四）   —— Nacos 配置 + Redis 缓存
★★★☆☆ 并行学（阶段五）     —— 基本的 DDD 分层理解
★★☆☆☆ 按需学（阶段六）     —— RabbitMQ 消息收发
★☆☆☆☆ 后面再学             —— ES/Talos/FDS/Spring Cloud 微服务
```

## 建议的学习节奏

- **每天学习时间**：建议连续，不要中断超过 2 天
- **学练比例**：1:1，看文档半小时就要写代码半小时
- **遇到问题怎么办**：先看团队现有代码里类似场景怎么写的，模仿是最高效的方式
- **快捷键**：IDEA 的 `Ctrl+Shift+T`（跳转测试/实现）、`Alt+Enter`（智能修复）必记

## 需要等项目代码后才能补充的内容

- 项目的 Maven 模块结构
- 现有的包目录组织
- 自定义的统一返回 `R<T>` 实现
- Nacos 上实际的 Data ID 和配置项
- RabbitMQ 交换机 / 队列的命名规范
- 团队约定的异常处理方式（全局 `@RestControllerAdvice`）
- 现有的 Docker Compose 或部署配置
